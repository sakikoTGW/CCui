# CCui 产品定义与界面设计（Product & UI Design）

> 配套：`GUI_PLAN.md`（路线/架构/市场依据）。本文回答三件事：**做什么 / 怎么设计 / 长什么样**。
> 设计准则遵循 ui-ux-pro-max：一屏一个主操作、2 秒可扫描、高频操作 ≤2 步、避免 AI 套路审美。

---

## 1. 产品定义

**一句话**：一个开源的 **AI 编码 Agent 驾驶舱**——把"指挥 agent、审查改动、扩展能力"做成看得见、控得住的桌面端。

| 维度 | 定义 |
|---|---|
| 是什么 | 桌面端（Electron）的 agent 控制台，复用现有 `src/` 引擎 |
| 给谁用 | ① 重度用 AI 编码的开发者 ② 小团队 ③ 爱折腾/想自定义扩展的人（开源社区）|
| 一句差异化 | 别人给 1 个聊天 agent；我们给"**多 agent 编排 + 改动可信可审 + 可无限扩展**"的驾驶舱 |

**边界——明确不做什么（防 feature sprawl）**
- ❌ 不做完整代码编辑器/IDE（不和 VSCode 抢编辑器；需要时跳到用户自己的编辑器）
- ❌ 不做云托管后端（本地优先，daemon 跑在本机；协作是后期可选层）
- ❌ 不自研模型（接 DeepSeek/Anthropic/兼容 API，主打多模型可切）

---

## 2. 版本切分（框架优先策略）

> **策略定调**：不做"功能完整但封闭的薄 MVP"，改为 **Walking Skeleton——先把整个框架骨架搭通，每个模块留好位置 + 最小实现，再逐期加厚。**
> 核心机制 **「模型分级路由」（强模型干难活、弱模型干简单活）是框架一等公民，从第一天就在**，不是后期优化。

### v0.1 — Walking Skeleton「全框架骨架 + 模型分级」
目标：**骨架全通、主线能跑、各模块有位可插**。

- **Core daemon + 事件总线 + 工具/权限**（复用 `src/`）——所有上层都订阅事件总线
- **ModelRouter（核心）**：按任务复杂度/类型/预算把 LLM 调用分派给 强/弱模型；规则可视化可改
- **调度骨架**：单 agent 先跑通，但接口按多 agent 设计；`coordinatorMode` = 强模型协调 + 弱模型执行（先最小实现）
- **前端骨架**：活动栏 6 区**全部建好**（多数 soon/最小态），对话屏可用 + 驾驶舱（含"本步用了哪个模型、花了多少"）
- **预留接口**：记忆 / 文件语义 / 编排 / 插件 / 协作 都留好挂载点与空壳，不实现细节

> 衡量标准：发一句话能跑通完整链路，且能看到"这步走强模型规划、那步走弱模型执行"，骨架就立住了。

### 后续：把空壳逐个加厚（骨架已通，每期是"填肉"非"动骨"）
- **v0.2** 记忆与自我改进（M3）+ 文件语义识别（M4）
- **v0.3** 多 agent 编排台（M5，护城河）+ 完整可观测拓扑
- **v0.4** 可扩展平台 + 插件市场（M6，Live2D/Unity 示例）
- **v0.5** 多人协作（M7）

## 3. 设计原则（落地到每一屏）

1. **一屏一个主操作**：对话屏的主操作是"输入/审查改动"，其余降权。
2. **2 秒可扫描**：进任意屏，2 秒内看到"我在哪 / 现在什么最重要 / 下一步点哪"。
3. **渐进披露**：高级配置默认折叠（抽屉/chip/次级面板），默认路径短。
4. **即时反馈**：每个动作后有 toast/内联状态/loading，再提示下一步。
5. **键盘优先**：Esc 关浮层、Tab 跟随视觉流、Cmd/Ctrl+K 命令面板直达一切。
6. **状态完备**：每个视图都设计 空 / 加载 / 错误 / 无权限 四态。

## 4. 信息架构（全局结构）

左侧**活动栏**（图标导航，VSCode 式），点击切换主区：

```
[💬] 对话 Chat        ← 默认，主战场
[🎛] 控制台 Console    ← skills/rules/mcp/权限
[🕸] 编排 Orchestrate  ← 多 agent（v0.3 解锁）
[📊] 可观测 Observe    ← trace/成本/回放
[🧩] 市场 Market       ← 插件（v0.4 解锁）
[⚙] 设置 Settings
```

未解锁的入口灰显并标 "soon"，让用户看见路线、不点空。

## 5. 主界面布局（对话屏 · v0.1 核心）

三栏：**活动栏（窄）｜ 对话区（主，最宽）｜ 驾驶舱（右侧，可收起）**

```
┌──┬───────────────────────────────────────┬───────────────────┐
│💬│  proj: e:\CCui          ● deepseek-v4   │  驾驶舱           │
│🎛│ ┌───────────────────────────────────┐ │ ───────────────── │
│🕸│ │ user: 给登录加错误处理            │ │ 上下文 ▓▓▓▓░ 62%  │
│📊│ │                                   │ │  sys 12% rules 8% │
│🧩│ │ assistant: 我先看相关文件…        │ │  files 30% hist…  │
│⚙ │ │  ┌─ 🔧 Read  src/auth.ts  120ms─┐ │ │ ───────────────── │
│  │ │  └───────────────────────────── │ │ 本轮 1.2k tok ¥0.03│
│  │ │  ┌─ ✏ Edit src/auth.ts ────────┐│ │ 累计 18k tok ¥0.41 │
│  │ │  │  + try { ... } catch {...}   ││ │ ───────────────── │
│  │ │  │  [✓接受] [✗拒绝] [查看全文]  ││ │ 工具时间线         │
│  │ │  └──────────────────────────────┘│ │  Read   ▌120ms    │
│  │ └───────────────────────────────────┘ │  Edit   ▌ 80ms    │
│  │ ┌───────────────────────────────────┐ │  Bash   ▌1.4s ⚠   │
│  │ │ 输入…           [@文件] [▶ 发送]  │ │ ───────────────── │
│  │ └───────────────────────────────────┘ │ 权限: 接受编辑 ▾  │
└──┴───────────────────────────────────────┴───────────────────┘
```

- **主操作**：底部输入框 + 行内 diff 卡片的"接受/拒绝"。其余（驾驶舱）是只读监控，视觉降权（更小字号/低对比）。
- 驾驶舱可一键收起 → 对话区全宽（专注模式）。
- 顶栏永远显示：当前项目 / 当前模型 / 连接状态（"我在哪"）。

## 6. 关键屏设计

### 6.1 Diff 审查卡片（v0.1 核心爽点）
- 工具产生的每处文件改动 = 一张内联卡片：文件名 + 行数变化 + 折叠的 diff
- 主操作：`✓接受 / ✗拒绝`；次操作：展开全文、在编辑器打开
- 多处改动时顶部提供 `全部接受 / 全部拒绝`，但**高风险**（删除/动 schema/动密钥）单独标红、禁用"全部接受"，强制逐条确认（落地 guardrails）

### 6.2 权限请求卡片
- 出现在对话流中（不是打断式 modal，可继续读上文）
```
┌─ ⚠ 需要授权 ─────────────────────────────┐
│ 运行 Bash:  rm -rf ./dist                 │
│ 风险: 删除文件                            │
│ [拒绝] [仅此次允许] [始终允许 Bash]        │
└───────────────────────────────────────────┘
```
- 三选项：拒绝 / 仅此次 / 始终允许（写回 settings）。"始终允许"是缺失已久的便捷开关。

### 6.3 控制台 Console（v0.1）
- 顶部 4 个 Tab：Skills ｜ Rules ｜ MCP ｜ 权限
- 列表行密度高（紧凑），每行：名称 + **来源徽章**（内置/项目/全局/插件）+ 开关 + 状态点
```
Skills                                      [+ 添加]
 ● caveman          [插件]   ▣ 开   就绪
 ● ui-ux-pro-max    [全局]   ▣ 开   就绪
 ○ sts2-modding     [全局]   ▢ 关
Rules
 ● ccui-dev.mdc     [项目]   ▣ 开
 ⚠ .cursor/rules    [项目]   未读取 → [导入]
```
- 关键补强：显式列出 `.cursor/rules` 并提供"导入"（现在完全不读）。

### 6.4 可观测 Observe（v0.1 基础 / v0.3 完整）
- v0.1：执行时间线 + 成本曲线（就是驾驶舱的放大版）
- v0.3：OTel span 树 + **会话回放滑块**（拖到任意步看当时状态）+ 多 agent 拓扑图

### 6.5 编排 Orchestrate（v0.3 预览）
- 看板：每个子 agent 一列（状态灯 + 当前工具 + 产出摘要）
- 主控操作：派活 / 暂停 / 接管 / 合并；顶部一条任务进度

## 8. 状态设计（每屏必做四态）

| 态 | 例（对话屏）|
|---|---|
| 空 | 首次进入：一句引导 + 3 个示例 prompt 按钮 + "选择项目"|
| 加载 | 流式时显打字态 + 可中断按钮；daemon 启动时进度条 |
| 错误 | 人话错误 + 可操作建议（如"API Key 失效 → 去设置"），非堆栈 |
| 无权限 | 工具被拒后，行内显示"已拒绝"+ 重新授权入口 |
